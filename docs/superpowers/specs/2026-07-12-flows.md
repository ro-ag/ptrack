# ptrack ‚Äî Flow Catalog

**Date:** 2026-07-12
**Purpose:** Enumerate the normal flows (happy paths + realistic edges) so the
v0.2.0 design and tests cover them. Each flow: **trigger ‚Üí expected behavior**.

## A. Setup / init (developer)

| # | Flow | Expected |
|---|---|---|
| A1 | `init` in a clean git repo root | Create `.ptrack/ptrack.db` at repo root; stamp format v1 + writer version; register in global; print db path. |
| A2 | `init` in a git **subdir** | Resolve to the git root; create there (not the subdir). One project per repo. |
| A3 | `init` in a subdir when a project **already exists** above | Refuse: `already inside ptrack project at <root>`; non-zero exit. `--here` to force a nested project anyway. |
| A4 | `init` in a non-git dir | Create `.ptrack` in cwd. |
| A5 | `init` in an **empty** dir | Create DB fine; if guidance-scan runs, note "empty project, nothing to analyze". No error. |
| A6 | `init` where `.ptrack/ptrack.db` already exists | Refuse: `ptrack project already exists at <path>`; non-zero. (Idempotent, no clobber.) |
| A7 | `init --goal "X"` | As A1 plus set goal in one step. |
| A8 | `init --root <dir>` | Create at the explicit dir, bypassing git-root resolution. |
| A9 | `init` then agent-guidance install | Write/refresh the "how to use ptrack" section into agent-instruction file(s); idempotent (marker-delimited). |

## B. Agent session lifecycle

| # | Flow | Expected |
|---|---|---|
| B1 | **Cold start** ‚Äî fresh agent resumes | `ptrack context` ‚Üí bounded digest: goal, summary, active plan + open tasks, blocked tasks, recent notes, inventory + drill commands. |
| B2 | Agent needs the one next action | `ptrack next` ‚Üí single most-actionable task (active plan: doing else todo). |
| B3 | Agent drills into a plan/task | `plan show <id>` / `task show <id>` ‚Üí full detail + attached notes. |
| B4 | Agent records a decision | `note add "..." --task N` (or --plan / project). |
| B5 | Agent advances work | `task start/done/block <id>`; `plan done/use <id>`. |
| B6 | **Wind down** ‚Äî session ending | `summary set "where we are"` so the next cold start reloads it. |
| B7 | Agent searches for prior context | `search <term>` across plan/task titles + note bodies. |
| B8 | Agent parses programmatically | any read command `--json`. |

## C. Query / read (bounded payload)

| # | Flow | Expected |
|---|---|---|
| C1 | `context` on a huge project | Payload stays ~constant: counts + pointers, not full dumps. |
| C2 | `task list --status doing,blocked` | Only actionable tasks. |
| C3 | `note list --plan N --limit K` | Newest-first, scoped, bounded. |
| C4 | `plan list` | One line per plan + open/done counts. |
| C5 | `status` | Goal line, active plan, task counts. |
| C6 | Read command, no project | Clear `no ptrack project (run 'ptrack init')`; non-zero. |
| C7 | `show`/status query with unknown id | `plan N not found`; non-zero. |

## D. Mutation

| # | Flow | Expected |
|---|---|---|
| D1 | `plan add` | New active plan, incremental id, printed. |
| D2 | `task add` with no `--plan` and an active plan set | Attach to active plan. |
| D3 | `task add` with no `--plan` and **no** active plan | Error: set one or pass `--plan`. |
| D4 | `task add --plan N` where N missing | `ErrNotFound`; non-zero. |
| D5 | `plan use N` | Set active plan (verifies existence). |
| D6 | status transitions | todo‚Üîdoing‚Üîdone‚Üîblocked all reachable; timestamps update. |
| D7 | `note add` to task/plan/project | Stored with correct target + id. |

## E. Versioning / migration

| # | Flow | Expected |
|---|---|---|
| E1 | Open a DB with **no** format stamp (v0.1.0 DB) | Adopt as v1, stamp in place transparently. |
| E2 | Open a DB with format == CurrentFormat | Normal. |
| E3 | Open a DB with format **<** current | Run ordered migrations up to current, restamp; then proceed. |
| E4 | Open a DB with format **>** current (written by newer ptrack) | Refuse: `database format vN newer than this ptrack (supports vM) ‚Äî upgrade`; non-zero. Do NOT write. |
| E5 | `version` / `--version` | Print ptrack semver (ldflags, else build info, else `dev`). |
| E6 | Any write | Record `LastWriteVersion` (informational). |

## F. Multi-project / registry

| # | Flow | Expected |
|---|---|---|
| F1 | Work across several projects | Each cwd resolves to its own `.ptrack`; global registry tracks LastSeen. |
| F2 | `projects` | List registered projects, most-recent first. |
| F3 | Registry entry whose dir was deleted | Listed but flagged/soft ‚Äî never crashes a query. (Nice-to-have.) |
| F4 | `PTRACK_HOME` override | Global store + backups relocate; used by tests and sandboxes. |

## H. TUI (human)

| # | Flow | Expected |
|---|---|---|
| H1 | Bare `ptrack` inside a project | Launch dashboard on the active plan. |
| H2 | Bare `ptrack` **outside** a project | Return `ErrNoProject` before opening the UI; printed to stderr. |
| H3 | Add/edit via keys | plan/task/goal/summary/note mutate store, reload. |
| H4 | Backup key | Same service as `ptrack backup`. |

## I. Error / concurrency edges

| # | Flow | Expected |
|---|---|---|
| I1 | Two ptrack processes on one DB | bbolt file lock (1s timeout) ‚Üí second gets a clear "database in use" error, not corruption. |
| I2 | Corrupt / unreadable DB | Wrapped error naming the path; non-zero. |
| I3 | Read-only filesystem on init | Clear error, no partial `.ptrack`. |
| I4 | Every failure path | Non-zero exit + message to stderr (agents detect via exit code). |

## J. Kanban board (view over task status ‚Äî no schema change)

Columns: **Todo ¬∑ Doing ¬∑ Blocked ¬∑ Done** (blocked mid-pipeline = visible stuck work).

| # | Flow | Expected |
|---|---|---|
| J1 | `board` in a project with an active plan | Group the active plan's tasks into 4 status columns; Markdown sections (or `--json` object keyed by status). |
| J2 | `board --plan N` | Board for plan N (ErrNotFound if missing). |
| J3 | `board` with no active plan and no `--plan` | Error: pick a plan or set an active one. |
| J4 | `board --all` | Group by plan, each plan its columns; caution/bounded on large projects (counts + pointers if truncated). |
| J5 | `board --json` | `{ "todo": [...], "doing": [...], "blocked": [...], "done": [...] }` (or per-plan map for `--all`). |
| J6 | Empty column | Rendered as `_none_` / empty array, never omitted (stable shape). |
| J7 | TUI: toggle list‚áÑboard with `v` | Board view with 4 columns. |
| J8 | TUI: move card with `‚Üê/‚Üí` (`h/l`) | Selected task's status changes to the target column via `SetTaskStatus`; reload. |
| J9 | TUI: `‚Üë/‚Üì` in board | Navigate within the focused column; wrap/clamp sanely. |

## Coverage note

Flows marked as edges (A3, A5, C6/C7, D3/D4, E1/E3/E4, H2, I1) become explicit
test cases. E4 (reject newer format) and A3 (no accidental nested project) are
the two highest-risk correctness flows.
